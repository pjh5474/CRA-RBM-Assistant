# 포트폴리오 해석 (Portfolio Interpretation)

## 1. 커리어 전환 맥락

이 프로젝트는 데이터 팀 경력 3년과 소프트웨어 개발 경험이 CRA 모니터링 워크플로에 어떻게 적용될 수 있는지 보여 주기 위해 만들었습니다.

## 2. 핵심 메시지

임상시험 모니터링 활동을 구조화된 데이터 워크플로로 옮깁니다. 여기에는 위험 검토, 이슈 추적, 필수 문서 준비 상태, 프로토콜 일탈 추적, 동의서(ICF) 버전 점검, 모니터링 보고서 초안 생성이 포함됩니다.

## 3. 보여 주는 강점

- 데이터 품질 관리
- 상태(status) 기반 추적
- 이슈 분류
- API·데이터베이스 설계
- 감사(audit) 유사 추적성
- 대시보드 기반 검토 워크플로
- CRA 지향 문서 작성 지원

## 4. 기능 ↔ CRA 업무 매핑

| 기능                       | CRA 업무                         | 보여 주는 나의 강점                           |
| -------------------------- | -------------------------------- | --------------------------------------------- |
| Risk Dashboard             | 사이트 위험 우선순위화           | 데이터 지표 설계                              |
| Essential Document Tracker | 사이트 파일(필수 문서) 준비 상태 | 누락·만료 상태 관리                           |
| Protocol Deviation Tracker | 일탈 후속 조치                   | 이슈 분류 및 CAPA(시정·예방 조치) 스타일 추적 |
| ICF Version Check          | 동의서 버전 검토                 | 날짜·버전 일치 검증                           |
| Monitoring Report Draft    | 모니터링 문서화                  | 구조화된 데이터→문서 워크플로                 |
| Audit-like Logs            | 변경 추적성                      | 로그 기반 이슈·변경 추적                      |
| Auth-gated Import          | 통제된 쓰기 작업                 | 배포 환경을 고려한 서비스 설계                |
